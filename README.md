# Proyecto-tema-2-TIC
El cliente quiere una aplicación web sencilla para organizar tareas. Desea que cada usuario pueda crear, editar y marcar tareas como completadas, y que la información quede guardada. Además, pide que sea fácil de usar y funcione bien en móviles. Más adelante le gustaría tener un panel para ver el progreso de todos los usuarios.

**Metodología**
Scrum, porque permite trabajar por fases cortas llamadas sprints, hacer entregas frecuentes y adaptarse fácilmente a los cambios que el cliente pueda pedir. Además, es una metodología ideal para grupos pequeños o incluso para trabajar en solitario, ya que facilita la organización y la planificación del trabajo.

**Etapas del ciclo de vida del proyecto, roles y miembros del grupo**
El ciclo de vida del proyecto con Scrum incluye varias etapas: 
La fase de inicio, donde se definen los objetivos y se crea el Product Backlog; la planificación del sprint, donde se eligen las tareas a realizar; la ejecución, donde se desarrolla y prueba el producto; la revisión del sprint, donde se muestra el progreso al cliente; y la retrospectiva, donde se analizan los resultados y se proponen mejoras. 
Los roles principales son el Product Owner, encargado de representar al cliente; el Scrum Master, que organiza el trabajo y elimina obstáculos; y el equipo de desarrollo, que se encarga de programar y documentar. 
Las reuniones más importantes son el Daily Scrum (diaria), la Sprint Planning (al inicio del sprint), la Review (para mostrar el trabajo al cliente) y la Retrospectiva (para mejorar el proceso).

**Requisitos funcionales y no funcionales**
-Requisitos funcionales: permitir a los usuarios registrarse e iniciar sesión, crear, editar y eliminar tareas, marcar tareas como completadas, mostrar las tareas filtradas por estado y permitir que un administrador vea todas las tareas y usuarios. 
-Requisitos no funcionales: que la aplicación sea responsive, segura, rápida, con una interfaz sencilla e intuitiva, y que el código esté bien documentado y versionado.

**Herramientas necesarias**
Las herramientas necesarias para desarrollar el proyecto serán: Git y GitHub para el control de versiones, Visual Studio Code como IDE, HTML, CSS, JavaScript y Node.js como lenguajes de programación, Express.js y Bootstrap como frameworks, MongoDB como base de datos, Markdown y GitHub Wiki para la documentación, Jest para las pruebas y GitHub Projects o Trello para la gestión ágil de tareas y sprints.

**Plan de desarrollo**
Semana 1: Creación del repositorio y estructura inicial y definición del Product Backlog con el cliente.
Semana 2-3 (Sprint 1): Implementar registro e inicio de sesión y crear CRUD básico de tareas.
Semana 4-5 (Sprint 2): Implementar panel de administrador y mejorar diseño y usabilidad.
Semana 6 (Sprint 3): Pruebas y corrección de errores y documentación final y despliegue en GitHub Pages o Vercel.
Semana 7:Revisión final con el cliente y entrega.
